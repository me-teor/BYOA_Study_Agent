# AI Code Review

Code review helps improve code quality, detect defects, and share knowledge within a team.

## What Code Review Should Catch

A good code review should focus on:

1. Logic and correctness errors.
2. Readability and maintainability.
3. Performance issues.
4. Security problems.
5. Codebase-specific best practices and idioms.

## Good Review Comments

A good review comment should be specific, reference concrete code, explain the problem, and suggest a possible solution.

## AI Code Review

AI code review tools can improve efficiency, consistency, and early bug detection. They can reduce cognitive load by checking routine issues.

## Limitations

AI review tools may produce false positives, miss complex business logic, misunderstand repository-specific conventions, and fail to catch subtle edge cases. Humans still own the code that is merged and shipped.
